# Improving Tips

Scan existing tips for:

- bad/inconsistent categories or duplicate teaching points
- misleading details, or tips too broad/overloaded
- setup or config-authoring advice that belongs in docs, not the tip set
- summaries leading with a plugin/option name when the user outcome is clearer
- label-style summaries, mixed pair phrasing, mode-name drift, formatting noise
- dash/colon/paren key separators in summaries — keys attach with a space only
  (search the sources for ` - ` to catch strays)
- bare text objects shown as commands (`Select a class ac`) — a keystroke in the
  summary that does nothing when typed; pair with an operator (see [Every
  keystroke shown must do something](display-and-wording.md#every-keystroke-shown-must-do-something-when-typed))
- tips that only make sense after another (random order breaks them) — fold into
  the host tip or split (see [Each tip stands alone](display-and-wording.md#each-tip-stands-alone-display-order-is-random))
- tips to merge, split, retag, or delete

Use the [Wording table](display-and-wording.md#wording-good-vs-worse) and
[Categories](categories.md) rules. Remember the summary-hash
[identity](display-and-wording.md#display) cost before rewording.
